# ON-AIR 웹앱 점검 보고서

작성일: 2026-06-18
대상: `index.html` (단일 파일 SPA) · `manifest.json` · `sw.js`
기준: 엔터프라이즈(대기업) 웹앱 품질 — 기능·보안·개인정보·접근성(WCAG)·성능·PWA·반응형·코드품질

---

## 1. 총평

기능 완성도와 디자인 시스템(다크모드, CSS 변수, 기기 스킨)은 개인 프로젝트 기준으로 매우 우수합니다. 다만 "대기업 웹앱" 기준으로 보면 **접근성(A11y), 반응형, 보안 하드닝(CSP)** 세 영역이 가장 큰 보완 대상입니다. 치명적(서비스 불능) 결함은 없으며, 대부분 "표준 준수 강화" 성격입니다.

| 영역 | 평가 | 핵심 이슈 |
|---|---|---|
| 기능/버그 | 양호 | 키 없는 DJ추천 동작 안 함, 모달 안내문 불일치 |
| 보안 | 보통 | CSP 부재, 인라인 핸들러, 외부링크 rel 누락 |
| 개인정보 | 양호 | 전부 로컬 저장(추적 없음), 유튜브 쿠키만 유의 |
| 접근성 | **미흡** | aria-label·키보드 조작·모션 민감·대비 부족 |
| 성능 | 양호 | 일시정지 중 폴링, 이미지 lazy 미적용 |
| PWA | 우수 | manifest `id`·스크린샷 보강 권장 |
| 반응형 | **미흡** | 미디어쿼리 0개, 모바일 탭/스테이지 |
| 코드품질 | 보통 | 단일파일·인라인 핸들러·테스트 없음 |

종합: **B (양호, 표준 준수 보강 필요)**

## 2. 심각도별 요약

### High (우선 처리 권장)
1. **접근성 — 아이콘 버튼에 접근 이름 없음.** 재생·이전·다음·셔플·반복·즐겨찾기·가사·다크모드 버튼이 아이콘만 있고 `title`만 부여됨. 스크린리더 호환을 위해 `aria-label` 필요. 장식용 `<svg>`에는 `aria-hidden="true"` 권장.
2. **접근성 — 진행바(seek) 키보드 조작 불가.** `.track`(line 344)이 `onclick`만 있는 `div`. `role="slider"`, `tabindex`, 방향키 핸들러, `aria-valuenow` 부재. 키보드 사용자가 탐색 불가.
3. **접근성 — 모션 민감(전정기관) 대응 없음.** LP·CD 회전, 스피커 펄스 등 무한 애니메이션에 `@media (prefers-reduced-motion: reduce)` 미적용.
4. **접근성 — 색 대비 미달.** `--faint(#9ca3af)` 텍스트가 흰 배경에서 약 2.5:1로 WCAG AA(4.5:1) 미달. 힌트·플레이스홀더·부가정보 텍스트 해당.
5. **반응형 — 미디어쿼리 전무.** 모바일 전용 분기 0개. 탭 바(5개+버튼)가 좁은 화면에서 넘침/찌그러짐 가능(`.tabs`에 `overflow`/`wrap` 없음). 일부 스킨(가방형 330px)이 좁은 폰에서 가로 넘침.
6. **보안 — CSP(Content-Security-Policy) 부재 + 인라인 핸들러.** `onclick` 등 인라인 이벤트가 다수라 엄격한 CSP 적용이 막혀 있음. 대기업 표준에선 CSP 필수에 가깝고, 이를 위해 인라인 핸들러 제거가 선행돼야 함.

### Medium
7. **버그 — 키 없는 'DJ 추천'이 사실상 동작 안 함.** `runDJ()`(line 719)가 API 키 없을 때 `DEMO`를 정렬해 재생하는데 `DEMO=[]`(데모곡 제거됨)라 빈 큐로 끝나고, 메시지만 "골랐어요"라고 표시. 키 없는 사용자는 DJ추천이 먹통으로 보임.
8. **일관성 — 모달 안내문 불일치.** 키 설정 모달(line 385)에 "키 없이도 데모 곡·태그 추천은 됩니다"라고 적혀 있으나 데모곡이 제거되어 더 이상 사실이 아님.
9. **보안 — 외부 링크 rel 누락.** 모달의 '발급 페이지 열기'(line 385) `target="_blank"`에 `rel="noopener noreferrer"` 없음(탭내빙·referrer 누수 경미). 스테이지 오류 링크는 적용돼 있음.
10. **개인정보 — 유튜브 쿠키 도메인.** `youtube.com` iframe 사용. 추적 최소화를 위해 `youtube-nocookie.com`(개인정보 강화 모드) 권장.
11. **보안/운영 — API 키 취급.** 사용자가 입력한 YouTube Data API 키가 localStorage 평문 저장 + 요청 URL 쿼리로 전송(클라이언트 사용 특성상 불가피). Google Cloud에서 **HTTP 리퍼러 제한** 설정 안내를 노출하면 키 도용 위험을 낮출 수 있음.
12. **UX/상태 — 재생목록(큐) 미저장.** 즐겨찾기·보관함·최근·플레이리스트는 저장되나 현재 큐는 새로고침 시 사라짐(세션 한정). 의도라면 OK, 아니면 보관 권장.
13. **성능 — 일시정지 중에도 진행 폴링 지속.** `startPoll`(0.5s setInterval)이 PAUSED에서 정지되지 않음. `onState` PAUSED에서 `clearInterval(poll)` 추가 시 소폭 절약.
14. **복원력 — 전역 에러 핸들러 없음.** `window.onerror`/`unhandledrejection` 로깅 없음. 운영 중 오류 가시성 낮음.
15. **PWA — manifest 보강.** `id`(앱 식별자), `screenshots`(설치 UI 향상) 누락. 아이콘이 `"purpose":"any maskable"` 단일 지정 — 플랫폼에 따라 과크롭 가능, `any`와 `maskable`을 분리 등록하는 편이 안전.

### Low
16. **localStorage 키 네이밍 혼재** — `aria_*`와 `onair_*` 혼용(유지보수성).
17. **이미지 지연로딩 미적용** — 목록 썸네일 `<img>`에 `loading="lazy"`/`decoding="async"` 권장.
18. **포커스 표시(focus-visible) 커스텀 없음** — 키보드 포커스 링이 브라우저 기본에 의존.
19. **라이브 영역 미설정** — 채팅 로그/Now Playing 갱신이 `aria-live`로 안내되지 않음.
20. **공유 메타 부재** — Open Graph/Twitter Card, canonical 없음(공유 미리보기 품질).
21. **빌드/품질 파이프라인 없음** — 린트·포맷·테스트·번들 없음(단일 파일 특성상 선택).

## 3. 카테고리별 상세

### 3.1 기능 / 버그
- 키 없는 DJ추천 무동작(#7) — 키 없을 땐 "링크 붙여넣기/키 등록"을 안내하거나, 데모 셋을 다시 두는 방향 택1.
- 모달 안내문 현행화 필요(#8).
- 임베드 제한 영상 처리(자동 스킵 + '유튜브에서 보기')와 링크 중복 방지, 보관함 oEmbed 메타, 전체 초기화 등은 정상 구현 확인.
- 큐 비영속(#12)은 사용자 기대와 다를 수 있어 명시 또는 저장 권장.

### 3.2 보안
- **XSS: 낮음(양호).** 목록/캐셋/아이팟 텍스트는 `esc()`로 이스케이프, Now Playing은 `textContent` 사용, `dec()`는 `<textarea>.value`로 안전 디코딩. 사용자/외부 데이터가 원시 innerHTML로 들어가는 경로 미발견.
- **CSP/인라인 핸들러(#6)** — 가장 큰 하드닝 포인트.
- **외부 링크 rel(#9)**, **SRI** — Google Fonts·YT API는 동적이라 SRI 부적합 → CSP로 대체 권장.
- **시크릿** — 코드 내 하드코딩 키 없음(양호). 키는 사용자 브라우저에만 저장.

### 3.3 개인정보
- 모든 사용자 데이터 로컬(localStorage) 저장, 자체 분석/추적 없음 — 우수.
- 외부 호출: YouTube iframe/API, fonts.googleapis, api.lyrics.ovh, youtube oEmbed. 유튜브 임베드 쿠키만 유의(#10).
- `전체 초기화`가 ON-AIR 키만 선별 삭제(같은 github.io 계정 타 프로젝트 보호) — 올바른 처리.

### 3.4 접근성 (WCAG 2.1 AA 관점)
- 인지 가능: 색 대비 미달(#4), 아이콘 의미 텍스트 부재(#1).
- 운용 가능: 진행바 키보드 불가(#2), 포커스 표시 약함(#18), 모션 민감(#3).
- 견고성: 아이콘 SVG에 `aria-hidden` 없음, 동적 갱신 `aria-live` 없음(#19).
- 잘된 점: `lang="ko"`, 볼륨은 표준 `range` 입력, 시맨틱 `<header>/<button>` 사용, 단축키 제공.

### 3.5 성능
- 폰트 preconnect + `display=swap` 적용(양호).
- 일시정지 폴링(#13), 썸네일 lazy 미적용(#17) 외 큰 병목 없음. 단일 파일이라 추가 요청 적음.
- 무한 애니메이션이 저사양/배터리에 부담 가능 — `prefers-reduced-motion`로 동시 해결 권장.

### 3.6 PWA
- manifest: name/short_name/icons(192·512 maskable)/theme/bg/standalone/scope/start_url 충실 — 우수. 보강은 #15.
- service worker: 페이지 네트워크 우선 + 자산 캐시 우선, `skipWaiting`+`clients.claim`, 캐시 버전 관리(onair-v2) — 모범적. 오프라인은 앱 셸까지(재생은 네트워크 필요 — 정상).
- iOS 메타(apple-touch-icon, status-bar 등) 구비.

### 3.7 반응형 / 모바일
- 미디어쿼리 0개(#5)가 핵심. 모바일에서 탭 바 넘침, 일부 스킨 가로 넘침 가능.
- `viewport-fit=cover`로 노치 대응은 되어 있음.
- 스테이지 고정 높이(340px)는 작은 화면에서 비중이 큼 — 가변 높이/clamp 검토.

### 3.8 코드 품질 / 유지보수
- 단일 HTML에 CSS·JS 인라인. 규모 확장 시 파일 분리·모듈화·번들·린트·테스트 도입 권장.
- ES5 스타일(var/function)로 호환성 넓음. 전역 변수 다수(네임스페이스화 여지).
- localStorage 키 네이밍 혼재(#16).

### 3.9 SEO / 메타 / 호환성
- title·description·lang 구비. OG/Twitter/canonical 부재(#20).
- 최신 브라우저 대상(fetch/Promise/classList) — IE 미지원이나 2026년 기준 무방.

## 4. 우선순위 권장 로드맵

**1단계 (빠른 수정 · 반나절)**
- 아이콘 버튼 `aria-label` + 장식 SVG `aria-hidden` 부여 (#1)
- `--faint` 대비 상향, `:focus-visible` 링 추가 (#4, #18)
- `@media (prefers-reduced-motion: reduce)`로 애니메이션 정지 (#3)
- 모달 외부 링크 `rel="noopener noreferrer"` (#9), 모달 안내문 현행화 (#8)
- 키 없는 DJ추천 동작 정리 (#7)
- 탭 바 `overflow-x:auto` 등 모바일 임시 보강 (#5 일부)

**2단계 (표준화 · 1~2일)**
- 미디어쿼리 기반 모바일 레이아웃 (#5)
- 진행바 `role="slider"` + 키보드 (#2)
- `youtube-nocookie.com` 전환 (#10), 일시정지 폴링 정지 (#13)
- manifest `id`/screenshots, 아이콘 purpose 분리 (#15)
- `aria-live`로 Now Playing/채팅 안내 (#19), 썸네일 lazy (#17)

**3단계 (하드닝/운영 · 선택)**
- 인라인 핸들러 제거 → 엄격 CSP 적용 (#6)
- 전역 에러 핸들링/로깅 (#14)
- 파일 분리·린트·간단한 e2e 테스트, 큐 영속화 (#12), 키 네이밍 통일 (#16)

## 5. 강점 (유지할 것)
- 일관된 디자인 토큰(CSS 변수) + 자연스러운 다크모드 전환
- 견고한 오류 UX: 임베드 제한 → '유튜브에서 보기', 자동 다음 곡, 빈 상태 안내 커버
- XSS 방어(이스케이프/textContent) 기본기 양호
- 모범적 PWA(매니페스트 + 네트워크 우선 SW + 오프라인 셸 + 아이콘)
- 개인정보 친화(로컬 저장, 추적 없음)와 선별적 전체 초기화
- 링크 중복 방지, 보관함 자동 메타(oEmbed), 키보드 단축키 등 디테일
